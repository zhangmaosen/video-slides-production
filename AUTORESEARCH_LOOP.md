# 幻灯片 Autoresearch Loop

## 核心理念

给 AI Agent 一个固定的工作流程，让它自主实验迭代。你早上醒来时，看到一串实验记录和（希望）更好的结果。

## 流程

```
┌─────────────────────────────────────────────────────────┐
│                    实验循环（每个 slide）                  │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│  1. 仓颉整理（读取 project_config + slides_content）     │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│  2. 女娲生成 prompt                                    │
│      - 读取 project_config.json (style)                 │
│      - 读取 slides_content.json (内容)                  │
│      - 读取 assets/ (参考图)                           │
│      - 输出: prompts/slide_XX/vN_*.txt                │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│  3. 哪吒生成图片                                      │
│      - 运行: python3 scripts/core/gen_slide.py         │
│      - 输出: slides/slide_XX_vN.png                   │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│  4. 二郎神评分                                         │
│      - 读取生成的图片                                   │
│      - 评分标准（100分）：                              │
│        - 标题准确性（一票否决）                         │
│        - 标题内容：10分                                │
│        - 故事性：25分                                  │
│        - 参考物神似度：30分                              │
│        - 视觉吸引力：15分                               │
│        - 风格一致性：10分                               │
│        - 其它文字：10分                                 │
│      - 通过标准：92分+                                 │
└─────────────────────────────────────────────────────────┘
                           │
              ┌────────────┴────────────┐
              │                         │
         >= 92 分                   < 92 分
              │                         │
              ▼                         ▼
┌─────────────────────┐     ┌─────────────────────────┐
│  5. 记录结果         │     │  5. 丢弃，继续优化      │
│  - 更新 CHANGELOG.md │     │  - 女娲优化 prompt      │
│  - git commit       │     │  - 回到步骤 2           │
│  - 下一张 slide      │     │  - 循环直到通过         │
└─────────────────────┘     └─────────────────────────┘
```

## 记录格式

每个 slide 的 `prompts/slide_XX/CHANGELOG.md`：

```markdown
# slide_00 Changelog

## v1 (2026-04-04)
- 初始版本
- 评分：75 分
- 问题：Tesla Logo 变形

## v2 (2026-04-04)
- 优化：修正 Tesla Logo 描述
- 评分：88 分
- 问题：文字融入效果差

## v3 (2026-04-04)
- 优化：改用 PIL 后处理文字
- 评分：92 分 ✅
- 状态：通过
```

## 关键约束

### 1. 固定通过标准
- **92 分** 是唯一标准
- 低于 92 必须继续优化
- 不能妥协

### 2. 只改 prompt，不改代码
- 哪吒只运行 `gen_slide.py`
- 不修改生成逻辑
- prompt 是唯一变量

### 3. 记录一切
- 每个版本都要记录
- 记录评分和问题
- git commit 每次迭代

### 4. 不停循环
- 一旦开始，不询问用户
- 继续迭代直到通过
- 用户可以随时打断

## 实验循环伪代码

```
for slide in slides:
    version = 1
    
    while True:
        # 1. 仓颉 + 女娲生成 prompt
        generate_prompt(slide, version)
        
        # 2. 哪吒生成图片
        generate_image(slide, version)
        
        # 3. 二郎神评分
        score = rate_image(slide, version)
        
        # 4. 记录
        log(slide, version, score)
        
        if score >= 92:
            git_commit(slide, version)
            break
        else:
            version += 1
            # 循环继续，女娲根据反馈优化
```

## 与 Karpathy Autoresearch 对比

| 维度 | Karpathy | 我们 |
|------|---------|------|
| 目标 | val_bpb 越低越好 | 评分 >= 92 |
| 变量 | train.py | prompt |
| 约束 | 固定 5 分钟 | 迭代次数 |
| 循环 | 无限 | 直到通过 |
| 记录 | results.tsv | CHANGELOG.md |

## 启动方式

```
开始新项目：
1. 仓颉读取 script.md → slides_content.json
2. 对每个 slide 启动实验循环

继续优化：
- 如果某个 slide 未通过，继续循环
- 如果中断，重新运行 gen_slide.py
```

## 停止条件

- 所有 18 张 slides 都达到 92 分
- 用户手动打断
