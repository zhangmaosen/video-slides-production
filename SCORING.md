# 评分机制 - 二郎神 subagent

## 核心原则

**每一步打分都用独立 subagent 来完成**

## subagent 名称：二郎神

**为什么用二郎神？**
- ✅ **天眼通**：能洞察图片的每一个细节
- ✅ **公正无私**：不受上下文干扰，独立客观评分
- ✅ **火眼金睛**：能发现 AI 生成的文字错误和特征偏差

## 评分流程

```
1. 生成 prompt → 2. AI 生成图片 → 3. 二郎神 subagent 评分 → 4. 如果<95 分，调整 prompt → 5. 重复直到 95 分+
```

## 二郎神评分任务模板

```python
sessions_spawn(
    runtime="subagent",
    mode="run",
    label="二郎神评分",
    task="""你是一个独立的图片评分专家，不受任何上下文影响。

请根据评分标准，对这张图片进行完整评分（100 分制），**必须和参考图片逐项对比**：

**待评分图片：** {image_path}

**参考图片：**
- {reference_images}

**逐字稿（必须严格对照）：**
{script_content}

**评分维度：**

1. **逐字稿遵循度（30 分）**
   - 主标题（15 分）：必须是"{main_title}"（不是其他任何变体）
   - 副标题（15 分）：必须是"{sub_title}"（不是其他任何变体）
   - 其他元素（15 分）：标注、数据、位置是否准确？

2. **故事性和自说明性（20 分）** - 无需文字能否理解核心观点？

3. **参考物神似度（20 分）- 必须和参考图片逐项对比**
   - {reference_checklist}

4. **技术准确性（15 分）** - 参数是否正确？

5. **视觉吸引力（10 分）** - 构图、色彩、冲击力如何？

6. **风格一致性（5 分）** - 是否有手绘感？

**请给出：**
- 每个维度的分数和详细理由
- 参考物神似度细分（0-10 分）
- 总分
- 是否达到 95 分目标

**注意：** 只根据图片内容客观评分，必须和参考图片逐项对比，必须和逐字稿逐字核对。"""
)
```

## 评分标准（100 分制）

| 维度 | 满分 | 说明 |
|------|------|------|
| **文字准确性** | **40** | **主标题 20 分 + 副标题/数据 20 分（一票否决）** |
| 故事性和自说明性 | 20 | 无需文字能否理解核心观点 |
| 参考物神似度 | 20 | 和参考图片逐项对比 |
| 视觉吸引力 | 10 | 构图、色彩、冲击力 |
| 风格一致性 | 10 | 是否符合目标风格 |
| **总分** | **100** | **通过标准：82 分+** |

**关键调整：**
- ✅ **文字准确性从 30 分提高到 40 分**（权重最高）
- ✅ **技术准确性合并到文字准确性**（数据也是文字）
- ✅ **风格一致性从 5 分提高到 10 分**
- ✅ **通过标准从 95 分调整为 82 分**（更务实的目标）

**文字准确性评分细则：**
- **主标题（20 分）**：
  - 如果逐字稿中有明确主标题 → 必须 100% 准确，错一个字直接 0 分
  - 如果逐字稿中无主标题 → **默认 20 分**（不扣分）
- **副标题/数据（20 分）**：
  - 如果逐字稿中有明确副标题/数据 → 必须 100% 准确，错一个字扣 5-10 分
  - 如果逐字稿中无副标题/数据 → **默认 20 分**（不扣分）
- **字体大小**：字体越大，出错概率越低，建议主标题 85px+，副标题 50px+

### 等级划分

| 等级 | 分数 | 说明 |
|------|------|------|
| S | 95-100 | 完美，可以直接使用 |
| A | 82-94 | **优秀，通过标准** |
| B | 70-81 | 良好，需要调整 |
| C | 60-69 | 及格，需要大改 |
| F | <60 | 不及格，重新生成 |

**通过标准：82 分+（A 级及以上）**

## 一票否决项（直接 F 级）

**前提：逐字稿中有明确要求**

- ❌ 主标题缺失/错误（仅当逐字稿中有明确主标题时）
- ❌ 副标题缺失/错误（仅当逐字稿中有明确副标题时）
- ❌ 关键数据错误（如价格、参数）
- ❌ 核心特征严重偏差（如车型错误）

**说明：**
- 如果逐字稿中没有明确要求主标题/副标题，则默认得分，不扣分
- 数据标注如果逐字稿中有，必须准确；如果没有，不强制要求

## autoresearch 优化流程

1. 生成图片 → 2. 二郎神评分 → 3. 如果<95 分，分析扣分项 → 4. 针对性调整 prompt → 5. 重复直到 95 分+

## Tesla Semi 项目经验

### 关键发现

- ✅ **文字准确性是一票否决项**：即使画面完美，文字错误直接 F 级
- ✅ **参考图片必须逐项对比**：不能凭印象评分
- ✅ **逐字稿必须逐字核对**：一个字都不能错
- ✅ **正向 + 负向提示词双管齐下**：v10 终于解决文字问题（30/30 分）

### 关键突破

**v10 使用正向 + 负向提示词双管齐下，终于解决了文字问题！**

**正向提示词：** 明确描述应该有什么
**负向提示词：** 明确排除不应该有什么

**示例负向提示词：**
```
- 数字渲染风格，平滑的数字渲染，无笔触感
- 银灰色车身，灰色车身，金属灰色
- 传统物理后视镜，大耳朵后视镜，粗壮单臂矩形镜面
- 房车比例，加高豪华客车，双层巴士
- 四扇独立小窗，垂直长方形小窗
- "绝美对比"，"路行"，"路线"，"完善生活舱"
- 文字错别字，文字乱码，模糊文字
```

## 最佳实践

1. **二郎神必须独立**：不受上下文干扰，只看图片和逐字稿
2. **逐字稿必须提供**：让二郎神逐字核对，一个字都不能错
3. **参考图片必须提供**：让二郎神逐项对比，不能凭印象
4. **82 分通过标准**：达到 82 分即可通过，继续优化到 95 分更好但不是必须
5. **正向 + 负向提示词双管齐下**：这是解决文字问题的关键
6. **记录每次评分**：建立优化历程，方便后续参考
7. **接受边际效益递减**：如果多次迭代都无法达到 82 分，考虑接受 B 级或手动修正
8. **字体放大**：字体越大，出错概率越低（主标题 85px+，副标题 50px+）
9. **减少文字**：不要每页都有标题，除非逐字稿明确；只保留核心数据
10. **逐字稿优先**：只有逐字稿中明确要求的文字才需要评分；如果没有要求，默认得分

## 常见问题

### Q: 为什么需要独立 subagent？
A: 避免上下文干扰，确保评分客观性。主 agent 可能已经对 prompt 有预设期望，影响评分公正性。

### Q: 为什么叫二郎神？
A: 二郎神有三只眼（天眼通），能洞察一切细节，公正无私，非常适合做评分工作。

### Q: 95 分目标是否太高？
A: 不，这是 S 级标准。Tesla Semi 项目证明，通过持续优化可以达到 95 分+。

### Q: 如果多次迭代都无法达到 95 分怎么办？
A: 考虑：
1. 接受 B 级（70-84 分），手动修正细节
2. 使用 PIL 添加文字（如果问题是文字）
3. 调整评分标准（如果某些维度确实无法达到）

### Q: 二郎神评分耗时多久？
A: 约 2-3 分钟（包括图片分析和逐字稿核对）
