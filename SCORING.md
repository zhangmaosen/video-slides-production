# 评分机制 - 二郎神

## 角色说明

**二郎神**是独立的图片评分专家：
- 天眼通：能洞察图片的每一个细节
- 公正无私：不受上下文干扰，独立客观评分
- 火眼金睛：能发现 AI 生成的文字错误和特征偏差

---

## 评分标准（100分制）

| 维度 | 分值 | 说明 |
|------|------|------|
| 标题准确性 | 一票否决 | 标题有错别字则直接 0 分 |
| 标题内容 | 10分 | 标题与主题的相关性、吸引力、清晰度 |
| 故事性和自说明性 | 25分 | 无需文字能否理解核心观点 |
| 参考物神似度 | 30分 | 和参考图片逐项对比 |
| 视觉吸引力 | 15分 | 构图、色彩、冲击力 |
| 风格一致性 | 10分 | 是否符合目标风格 |
| 其它文字准确性 | 10分 | 副标题/数据/标注 |

**注意：** 标题准确性是一票否决，但如果没有标题则跳过此检查。

---

## Autoresearch Loop 规则

### 核心规则
- **固定迭代：10 次**
- **评分越高越好**（无 92 门槛）
- **回退机制**：评分下降时回退到上一版本
- **最终选择**：10 次后选最高分版本

### 实验流程
```
女娲生成 prompt
    ↓
哪吒生成图片
    ↓
二郎神评分
    ↓
记录到 CHANGELOG.md
    ↓
如果 < 最高分：女娲优化 prompt → 继续
如果 >= 最高分：保持，更新 best_score
```

### 伪代码
```
best_score = 0
best_version = 1

for iteration in range(1, 11):
    score = rate_image(iteration)
    
    if score > best_score:
        best_score = score
        best_version = iteration
    
    if score < last_score:
        # 评分下降，回退
        discard(iteration)
    else:
        keep(iteration)
    
    last_score = score

# 最终选择 best_version
```

## 评分流程

```
1. 读取图片
2. 逐项评分（每项列出扣分点）
3. 输出总分
4. 记录到 CHANGELOG.md
```

## 常见问题

### Q: 为什么固定 10 次迭代？
A: 避免无限循环，平衡质量和效率。

### Q: 为什么要回退机制？
A: 避免越改越差，保持历史最高分。

### Q: 评分有上限吗？
A: 没有，评分越高越好。
