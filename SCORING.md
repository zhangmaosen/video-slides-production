# 评分机制 - 二郎神 subagent

## 核心原则

**每一步打分都用独立 subagent 来完成**

**完美主义者原则：不达 92 分不停止**
- 这是你的底线，也是你的骄傲
- 分数不达标就要一直做下去
- 绝不妥协，绝不跳过，绝不接受低于 82 分的质量

## subagent 名称：二郎神

**为什么用二郎神？**
- ✅ **天眼通**：能洞察图片的每一个细节
- ✅ **公正无私**：不受上下文干扰，独立客观评分
- ✅ **火眼金睛**：能发现 AI 生成的文字错误和特征偏差

## 评分流程

```
1. 生成 prompt → 2. AI 生成图片 → 3. 二郎神 subagent 评分 → 4. 如果<92 分，调整 prompt → 5. 重复直到 92 分+
```

**完美主义者执行规则：**
1. **不能妥协**：分数不达标就要继续优化，不能接受 B 级或 C 级
2. **记录历程**：每次迭代都要记录问题、评分、解决方案
3. **分析原因**：为什么没达标？是 prompt 问题、参数问题、还是其他？
4. **针对性优化**：根据扣分项调整 prompt，不盲目重试
5. **持续迭代**：v1 → v2 → v3... 直到达到 92 分+
6. **字体放大**：字体越大，出错概率越低
7. **减少文字**：不要每页都有标题，除非逐字稿明确

**标题准确性评分细则（一票否决）：**
- 如果逐字稿中有明确标题 → 必须 100% 准确，错一个字直接 **0 分**，整个评分作废
- 如果逐字稿中无标题 → **跳过此检查**，不影响评分
- **一票否决效果**：标题错则总分强制为 0，必须重新生成

**标题内容好坏评分细则（10 分）：**
- 如果有标题 → 评估标题与主题的相关性、吸引力、清晰度
- 如果无标题 → **默认 10 分**（不扣分）
- 扣分项：标题与主题不相关、标题平淡无吸引力、标题模糊难懂

**其它文字准确性评分细则（10 分）：**
- 如果逐字稿中有明确副标题/数据/标注 → 必须 100% 准确，错一个字扣 2-5 分
- 如果逐字稿中无副标题/数据/标注 → **默认 10 分**（不扣分）
- **字体大小**：字体越大，出错概率越低，建议主标题 85px+，副标题 50px+

### 等级划分

| 等级 | 分数 | 说明 |
|------|------|------|
| S | 95-100 | 完美，可以直接使用 |
| A | 82-94 | **优秀，通过标准** |
| B | 70-81 | 良好，需要调整 |
| C | 60-69 | 及格，需要大改 |
| F | <60 | 不及格，重新生成 |

**通过标准：82 分+（A 级及以上）**

## 一票否决项（直接 F 级）

**前提：逐字稿中有明确要求**

- ❌ 主标题缺失/错误（仅当逐字稿中有明确主标题时）
- ❌ 副标题缺失/错误（仅当逐字稿中有明确副标题时）
- ❌ 关键数据错误（如价格、参数）
- ❌ 核心特征严重偏差（如车型错误）

**说明：**
- 如果逐字稿中没有明确要求主标题/副标题，则默认得分，不扣分
- 数据标注如果逐字稿中有，必须准确；如果没有，不强制要求

**完美主义者核心规则：**
- ✅ **82 分是底线**：不能接受 B 级（70-81 分）
- ✅ **不能妥协**：即使迭代 20 次、30 次，也要达到 82 分
- ✅ **记录一切**：每次评分、每个问题、每个解决方案都要记录
- ✅ **分析原因**：不能盲目重试，要针对性优化

## Tesla Semi 项目经验

### 关键发现

- ✅ **文字准确性是一票否决项**：即使画面完美，文字错误直接 F 级
- ✅ **参考图片必须逐项对比**：不能凭印象评分
- ✅ **逐字稿必须逐字核对**：一个字都不能错
- ✅ **正向 + 负向提示词双管齐下**：这是解决文字问题的关键
- ✅ **12 次迭代才达到 82 分**：slide_10 证明了完美主义的价值

### 关键突破

**v10 使用正向 + 负向提示词双管齐下，解决了文字问题！**

**正向提示词：** 明确描述应该有什么
**负向提示词：** 明确排除不应该有什么

**示例负向提示词：**
```
- 数字渲染风格，平滑的数字渲染，无笔触感
- 银灰色车身，灰色车身，金属灰色
- 传统物理后视镜，大耳朵后视镜，粗壮单臂矩形镜面
- 房车比例，加高豪华客车，双层巴士
- 四扇独立小窗，垂直长方形小窗
- "绝美对比"，"路行"，"路线"，"完善生活舱"
- 文字错别字，文字乱码，模糊文字
```

## 常见问题

### Q: 为什么需要独立 subagent？
A: 避免上下文干扰，确保评分客观性。主 agent 可能已经对 prompt 有预设期望，影响评分公正性。

### Q: 为什么叫二郎神？
A: 二郎神有三只眼（天眼通），能洞察一切细节，公正无私，非常适合做评分工作。

### Q: 82 分目标是否太低？
A: 82 分是 A 级标准，已经很优秀。但如果想追求完美，可以继续优化到 95 分（S 级）。

### Q: 如果多次迭代都无法达到 82 分怎么办？
A: **完美主义者绝不妥协！** 考虑：
1. 深入分析扣分项，找出问题根源
2. 针对性调整 prompt（不要盲目重试）
3. 使用 PIL 添加文字（如果问题是文字）
4. 调整画面构图或元素（如果是视觉问题）
5. 继续迭代，直到达到 82 分

### Q: 二郎神评分耗时多久？
A: 约 2-3 分钟（包括图片分析和逐字稿核对）

### Q: 为什么需要记录优化历程？
A: 建立知识库，方便：
1. 追踪每次扣分项和解决方案
2. 发现反复出现的问题
3. 总结最佳实践
4. 后续项目参考

### Q: 二郎神评分耗时多久？
A: 约 2-3 分钟（包括图片分析和逐字稿核对）
